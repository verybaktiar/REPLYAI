Checklist Pengujian - QA

Gunakan daftar ini untuk memastikan seluruh fitur berjalan dengan normal. Beri tanda centang jika sukses.

1. Pengujian Dasar (Smoke Test)

  • Login ke Dashboard admin berhasil.
  • Sidebar navigasi tampil sempurna di semua halaman.
  • Berpindah antar menu tidak ada error (404/500).

2. Pengujian Inbox & Handoff

Skenario A: Bot Menjawab

  • Langkah: Kirim pesan "Halo" dari Simulator/Instagram.
  • Ekspektasi: Bot membalas. Status Inbox: Bot Handling.

Skenario B: Takeover oleh CS

  • Langkah: Buka percakapan di Inbox > Balas manual "Halo kak".
  • Ekspektasi: Pesan terkirim, status Agent Handling. Chat user lagi -> Bot Silent.

Skenario C: Handback ke Bot

  • Langkah: Klik "Kembalikan ke Bot".
  • Ekspektasi: Status kembali Bot Handling. Chat user lagi -> Bot balas.

3. Pengujian Rules (Auto Reply)

  • Langkah: Buat rule baru: Keyword="promo", Response="Ada diskon". Test di Simulator.
  • Ekspektasi: Bot membalas "Ada diskon".

4. Pengujian Knowledge Base

  • Langkah: Upload PDF info produk. Tunggu proses. Test tanya info dari PDF.
  • Ekspektasi: Bot menjawab menggunakan konteks PDF.

5. Pengujian Data Kontak (CRM)

  • Langkah: Chat user baru masuk. Cek menu Data Kontak.
  • Ekspektasi: User muncul, Total Pesan = angka (bukan n/a).

6. Pengujian Quick Replies

  • Langkah: Buat Shortcut /siap. Buka Inbox, ketik /.
  • Ekspektasi: Muncul sugesti, klik -> teks terisi.

7. Pengujian Jam Operasional

  • Langkah: Set jam tutup sekarang. Test chat dari Simulator.
  • Ekspektasi: Balasan "Pesan di Luar Jam Kerja".
